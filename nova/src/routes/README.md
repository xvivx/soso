## 路由介绍

### PC端路由
 - 普通路由: 路由切换会有微微向上移动的动画
 - 弹窗路由: 路由切换和弹窗动效一致, 并且这种路由在手机端会显示成页面

### 移动端路由
这类路由有三种形态, 不同点在于:
  - 一级路由: 切换时会有微微放大效果, 并且这种路由出现在首页的导航里
  - 二级路由: 切换时从右到左进入, 从左到右退出, 二级路由的进入会推动一级路由退出, 和桌面端区别是布局不一样, 和弹窗路由区别是弹窗路由在桌面端是弹窗, 该路由在桌面端也是页面
  - 弹窗路由: 动效和二级路由一致, 这类路由只是盖在其他路由之上, 并不会导致其他路由退出, 同时嵌套路由可以保持状态, 比如#/a -> #/a/b, 两个路由会同时存在

动效规范遵循上面的规则, 值得一提的是Tab路由, /a -> /a/1 , /a/2, /a/3, 无论桌面端还是手机端都是只在Tab路由激活的第一次动画, 子路由直接跳转使用replace

### 路由跳转
使用`useNavigate`这个hook做路由跳转, 这个hook是加强过的, 比useHistory好用, 绝对不能使用useHistory

### 路由的位置
开发者可以自行决定路由插入的位置, 在routes里有定义好的Layout, 根据需要选择, 特别注意静态路由（不需要token就能请求数据）要放在合适的位置。弹窗路由是全局加载的, 但是仍然可以通过摆放位置做到隔离, 比如登陆需要在每个页面都能唤起, 要注册在根路由下, 比如钱包里的路由可以放在Dashboard里, 可以缩小它的可见范围, 根据情况灵活处理。
